# rosdistant-quiz-helper
Chrome-расширение: ИИ-ассистент для онлайн-тестов с OCR и локальной БД знаний. Интеграция: Claude, Groq, Grok, GigaChat.
Study Helper — расширение для браузера Chrome, упрощающее работу со школьными и университетскими тестами. Приложение анализирует вопросы на поддерживаемых образовательных сайтах, может распознавать текст на изображениях (OCR) и хранить ответы в локальной базе знаний. Для генерации подсказок расширение отправляет вопросы непосредственно выбранным пользователем ИИ-сервисам (Anthropic Claude, Groq, xAI Grok, GigaChat) по HTTPS с помощью их API-ключей. Все данные пользователей остаются на их устройствах – расширение не передаёт информацию ни своему разработчику, ни третьим лицам

Обзор (Overview)
Study Helper — расширение браузера, помогающее учащимся при выполнении онлайн-тестов и работе с учебниками. Оно работает локально и не использует серверов разработчика. Основные возможности расширения: обнаружение вопросов на образовательных порталах, извлечение текста из изображений (OCR), поиск ранее сохранённых ответов и запросы к ИИ-провайдерам для генерации подсказок. Благодаря локальному хранению ключей API и данных и конфиденциальной архитектуре расширение соответствует требованиям Chrome Web Store (см. раздел «Конфиденциальность» и «Permissions»).

Функции (Features)
Мультиязычная поддержка: Проект может работать с вопросами и учебными материалами на русском и английском языках.
ИИ-провайдеры: Поддержка нескольких ИИ-сервисов:
Anthropic Claude (GPT-подобный ИИ от Anthropic)
Groq (серверный ИИ-сервис)
xAI Grok (ИИ-бот от xAI)
Сбер GigaChat (облачный ИИ-сервис от Сбера)
Пользователь вводит собственный API-ключ для выбранного провайдера, и расширение отправляет запросы напрямую к этому сервису.
Локальная база знаний: Расширение сохраняет найденные ответы, настройки и результаты OCR в chrome.storage.local/IndexedDB. Это позволяет повторно использовать решения и не тратить трафик при повторном появлении похожих вопросов. Пользователь может вручную очищать базу или экспортировать её через меню расширения.
OCR для изображений: Пользователь может делать скриншоты вопросов или загружать фото из учебников; расширение использует OCR для извлечения текста из изображения. Распознанный текст индексируется и служит источником при поиске ответов.
Экспорт данных: Сохранённые ответы, распознанный текст и конвертированные учебники можно экспортировать, например, в формате PDF или JSON для офлайн-анализа.

Контент-скрипты (CS): Выполняются на страницах тестирования (например, rosdistant.ru). Они извлекают текст вопросов, обрабатывают графику (для OCR) и передают данные в фоновый скрипт.
Фоновый сервис-воркер (BG): Отвечает за взаимодействие с локальным хранилищем и ИИ. При получении текста вопроса из контент-скрипта он может:
Сохранить вопрос/ответ в локальной БД (если функция автосохранения включена).
Выполнить OCR на полученном изображении (если необходимо).
Направить запрос к выбранному ИИ-провайдеру. Все запросы идут по защищённому HTTPS, и ответы сразу возвращаются пользователю.
Локальное хранилище (DB): Использует chrome.storage.local и IndexedDB для хранения настроек расширения, API-ключей пользователя, сохранённых ответов и результатов OCR. Размер хранимых данных может быть большим (полученные документы, множество вопросов), поэтому требуется разрешение unlimitedStorage. Хранилище на устройстве пользователя — разработчик к нему не получает доступа.
ИИ-провайдеры: Anthropic, Groq, xAI, GigaChat. Расширение отправляет на их серверы только фрагменты вопроса (и при необходимости часть учебника). Каждый провайдер обрабатывает данные по своей политике конфиденциальности (которую пользователь подтверждает при создании аккаунта с API-ключом). Разработчик расширения не выполняет роли прокси и не видит содержимое запросов.

Разрешения (Permissions)
Расширение запрашивает только необходимые разрешения (принцип минимум прав).

Разрешение	Цель использования
storage	Позволяет использовать chrome.storage API для локального сохранения настроек, API-ключей, базы вопросов/ответов.
unlimitedStorage	Дает неограниченный объём хранения в chrome.storage.local и IndexedDB. Нужно для хранения больших учебников, результатов OCR и обширной базы знаний.
declarativeNetRequestWithHostAccess	Используется вместе с host_permissions для безопасного маршрутизации сетевых запросов расширения. Нужен для фильтрации или изменения запросов к разрешённым хостам с помощью chrome.declarativeNetRequest.
host_permissions	Позволяет взаимодействовать с указанными доменами. Расширение работает на страницах образовательных порталов (пример: *://*.rosdistant.ru/*) и обращается к API-провайдерам (например, https://api.anthropic.com/*, https://api.groq.com/*, https://api.x.ai/*, https://gigachat.devices.sberbank.ru/*). Это нужно для отправки запросов к этим сервисам. Host permissions дают возможность выполнять fetch и другие API на этих сайтах.

При необходимости можно добавить комментарии к manifest.json, поясняющие назначение каждого разрешения (например, "storage" // сохранить настройки и ответы). Избегаем широких шаблонов (*://*/*), чтобы не запрашивать лишние права.

Удалённый код: Расширение не загружает и не выполняет JavaScript с внешних сайтов. Весь код включён в пакет расширения (Manifest V3 запрещает RHC по умолчанию). Поэтому в декларации для Chrome Web Store можно указать, что разрешение «Удалённый код» (Remote Code) не используется.

Приватность и поток данных (Privacy & Data Flow)
Расширение следует принципу «сбор только необходимого» и не ведёт пользователей. Ниже схема потоков данных:

Локальное хранение: Все пользовательские данные (ключ API, текст вопросов, сохранённые ответы, результаты OCR) сохраняются только на устройстве. Расширение не передает их на свои серверы. Удаление расширения или явное очищение базы через интерфейс расширения удаляет эти данные из браузера.
Отправка данных: При формировании подсказки расширение отправляет текст вопроса (и при необходимости фрагмент учебника) напрямую на сервер выбранного AI-провайдера по HTTPS. Данные идут от браузера пользователя к облаку провайдера (без участия разработчика). Обработка данных провайдером описана в его собственной политике конфиденциальности.
Нет трекинга: Расширение не интегрирует сторонние сервисы аналитики или рекламы. Оно не собирает данные о действиях пользователя вне своей функциональности. Запросы идут только к явно указанным внешним сервисам (ИИ-провайдерам) и авторизуются пользовательским API-ключом.
Важно: Даже если данные обрабатываются локально (не уходят в интернет), политика конфиденциальности всё равно требует об этом сообщать. В нашей политике мы открыто заявляем, что все данные остаются у пользователя и никуда не передаются. Пользователь даёт согласие на передачу данных ИИ-провайдерам при вводе API-ключа (такова процедура этих сервисов).

Безопасность (Security)
Все внешние соединения выполняются по HTTPS.
API-ключ сохраняется в chrome.storage.local; он не синхронизируется между устройствами и не шифруется дополнительно (по усмотрению пользователя можно хранить ключи в безопасных менеджерах).
Расширение не содержит рекламных скриптов, не отслеживает пользователей и не использует сторонние cookie.
Мы соблюдаем принцип минимизации данных: передаём только то, что нужно для ответа (не сохраняем лишние user-agents, не логируем посетённые сайты и т.д.).
При обнаружении уязвимостей (например, с помощью GitHub Dependabot или ручного аудита) изменения публикуются в SECURITY.md. Пользователь может сообщать об уязвимостях через Issues или электронную почту разработчика.
